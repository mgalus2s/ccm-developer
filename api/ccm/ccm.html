<static> load(resources [, callback])

load resource(s) (js, css, json and/or data from server interface)

Parameters:
Name Type Argument Description
resources string | Array

resource(s) URLs
callback function <optional>

callback when all resources are loaded (first parameter are the results)
Returns:

results (only if all resources already loaded)

Type
*
Examples
// load ccm component with callback (local path)
ccm.load( 'ccm.chat.js', function ( component ) {...} );
// load ccm component without callback (cross domain)
ccm.load( 'http://akless.github.io/ccm-developer/resources/ccm.chat.min.js' );
// load javascript file (local path)
ccm.load( 'scripts/helper.js' );
// load jQuery UI (cross domain)
ccm.load( 'https://code.jquery.com/ui/1.11.4/jquery-ui.min.js' );
// load css file (local path)
ccm.load( 'css/style.css' );
// load image file (cross domain)
ccm.load( 'http://www.fh-lsoopjava.de/ccm/img/dialogbox.png' );
// load json file with callback (local path)
ccm.load( 'json/data.json', function ( data ) { console.log( data ); } );
// load json file with callback (cross domain), json file must look like this: ccm.callback[ 'ccm.chat.templates.min.json' ]( {...} );
ccm.load( 'http://akless.github.io/ccm-developer/resources/ccm.chat.templates.min.json', function ( data ) {...} );
// data exchange between client and server php interface with callback (cross domain), php file must use jsonp
ccm.load( [ 'http://www.fh-lsoopjava.de/ccm/php/greetings.php', { name: 'world' } ], function ( response ) { console.log( response ); } );

<static> store( [settings] [, callback])

use of a ccm datastore

Parameters:
Name Type Argument Description
settings ccm.types.settings <optional>

ccm datastore settings
callback function <optional>

callback when ccm datastore is ready for use (first parameter is the ready ccm datastore)
Returns:

ready ccm datastore (only if synchron)

Type
ccm.Datastore
Examples
// create empty ccm datastore using local cache (synchron)
ccm.store();
// create ccm datastore using local cache with initial datasets by json file (asynchron)
ccm.store( { local: 'templates.json' } );
// same example but cross domain, json file must look like this: ccm.callback[ 'ccm.chat.templates.min.json' ]( {...} );
ccm.store( { local: 'http://akless.github.io/ccm-developer/resources/ccm.chat.templates.min.json' } );
// create ccm datastore using clientside database IndexedDB (asynchron)
ccm.store( { store: 'chat' } );
// create ccm datastore using serverside database (asynchron), the file index.js must be an <i>ccm</i> compatible interface
ccm.store( { store: 'chat', url: 'ws://ccm2.inf.h-brs.de/index.js' } );
