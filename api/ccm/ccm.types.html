component

ccm component object

Below you see typically (but not all mandatory) properties. Most of these properties are set by ccm runtime environment.

Type:
  • namespace
Properties:
Name Type Description
index ccm.types.index

ccm component index
name ccm.types.name

ccm component name
version ccm.types.version

ccm component version number
config ccm.types.config

default configuration for own ccm instances
Instance function

constructor for creating ccm instances out of this component
init function

callback when this component is registered
instance function

creates an ccm instance out of this component
render function

creates and renders an ccm instance
instances number

number of own created ccm instances
Example
{
  index:     'chat-2.1.3',
  name:      'chat',
  version:   [ 2, 1, 3 ],
  config:    {...},
  Instance:  function () {...},
  init:      function ( callback ) { ...; callback(); },
  instance:  function ( config, callback ) {...},
  render:    function ( config, callback ) {...},
  instances: 0
}

config

ccm instance configuration

Below you see typically (but not mandatory) properties.

Type:
  • object
Properties:
Name Type Description
element ccm.types.element

ccm instance website area
html ccm.types.dependency

ccm datastore for html templates
style ccm.types.dependency

CSS styles for own website area

html classes for own website area

ccm datastore that contains the dataset for rendering

Example
{
  element: jQuery( '#container' ),
  html:    [ ccm.store, { local: 'templates.json' } ],
  style:   [ ccm.load, 'style.css' ],
  classes: 'ccm-chat_snow'
  store:   [ ccm.store, { url: 'ws://ccm2.inf.h-brs.de/index.js', store: 'chat' } ],
  key:     'test',
  lang:    [ ccm.instance, 'https://kaul.inf.h-brs.de/ccm/components/lang.js', {
    store: [ ccm.store, 'translations.json' ]
  } ],
  user:    [ ccm.instance, 'https://kaul.inf.h-brs.de/ccm/components/user.js' ]
}

dataset

ccm dataset

Every ccm dataset has a property 'key' which contains the unique key of the dataset. There are no conventions for other properties. They can be as they want.

Type:
  • object
Examples
{
  "key": "demo",
  "text": "Hello, World!",
  "value": "4711"
}
{
  "key": "my_first_video_rating",
  "likes": {                       // users which clicks like button
    "akless": true,
    "hunny84": true
  },
  "dislikes": {                    // user which clicks dislike button
    "negativguy": true
  }
}
{
  "key": "fruit_game_settings",
  "max_player": 2,
  "fruits": [
    {
      "name": "Apple",
      "points": 50
    },
    {
      "name": "Pear",
      "points": 30
    }
  ]
}

key

key of a ccm dataset (unique in the ccm datastore which contains the dataset)

Must be conform with the regular expression /^[a-z_0-9][a-zA-Z_0-9]*$/.

Type:
  • string | number
Examples
"test"
"_foo"
4711
"1_ABC___4711"
"123"
"_"
